# 広告制作ワークフローガイド（見返し用HTML）

スライドショー／カルーセル型広告を、企画から実装・レビューまで迷わず進めるための作業マニュアルを、静的HTMLにまとめたものです。

次回以降に別サービスの広告を作るとき、このHTMLを開けば

- 最初に何を考えるか
- どの順番で進めるか
- Claude Codeに何を頼むか
- Codexに何を見てもらうか
- 提出前に何を確認するか

がすぐに分かるよう設計しています。

---

## 目的

- スライドショー／カルーセル型広告の制作手順を、長期的に見返せるドキュメントにする
- Claude Code 用・Codex 用の汎用テンプレを、すぐコピーして使える状態にする
- 提出前のチェック項目を、抜け漏れなく確認できるようにする

---

## 開き方

1. このフォルダ（`見返し用HTML/`）を開く
2. `index.html` をダブルクリックして、ブラウザで開く

ビルド不要・サーバー不要。ローカルファイルだけで動きます。

> Windows でファイルパスをコマンドから開く場合：
> `start index.html`

---

## ファイル構成

```
見返し用HTML/
  index.html        ← メインHTML（このファイルを開く）
  css/
    style.css       ← スタイル定義
  js/
    main.js         ← サイドナビ・折りたたみ・コピー・検索など
  README.md         ← このファイル
```

外部CDN・外部フォントは使っていません。すべてローカルで完結します。

---

## 実装した機能

- **サイドナビ**: クリックで該当セクションへスムーススクロール。スクロール位置に応じて現在地をハイライト
- **ページ内検索**: 見出し・本文・テンプレ内文字を対象にインクリメンタル検索、ヒットしたセクションだけ表示・ハイライト
- **折りたたみ表示**: Claude Code / Codex の長いテンプレを開閉可能（初期状態は閉じる）
- **コピー機能**: 各テンプレに「テンプレをコピー」ボタン。コピー成功時にトースト表示
- **チェックリスト**: クリックでチェックON/OFF（一時的なメモ用、ページ再読込でリセット）
- **トップへ戻るボタン**: 400px以上スクロールすると右下に表示
- **モバイル対応**: 画面幅 960px 以下でサイドナビをドロワー化、右上ハンバーガーで開閉
- **印刷対応**: 印刷時はサイドナビ・操作ボタンを非表示、折りたたみを全展開
- **アクセシビリティ**: `aria-expanded` / `aria-current` / `aria-live` / キーボード操作 / `prefers-reduced-motion` 対応

## 使い方

### 通常の作業フロー

1. ブラウザで `index.html` を開く
2. 「全体フロー」で全体感を確認
3. 「サービス設計」「コンセプト設計」「スライド構成」で企画を固める
4. 「Claude Code 用テンプレ」を**コピー**して埋め、Claude Code に渡す
5. 実装後、「Codex 用テンプレ」を**コピー**して埋め、Codex に渡す
6. 「最終提出前チェックリスト」をクリックで埋めながら提出準備

### テンプレをコピーする方法

- 各テンプレの右上にある「**テンプレをコピー**」ボタンを押す
- ボタンが「コピー済み」に変わり、画面下に「コピーしました」と表示されたら成功
- コピーされた内容はそのままChatGPT/Claude Code/Codexに貼り付けて編集できます

### 印刷したい場合

ブラウザの印刷機能（Ctrl + P）から印刷すると、サイドナビ・操作ボタンが非表示になり、折りたたみが自動展開されたドキュメント用レイアウトで出力されます。

## 設計メモ

- React/Vue/Svelte などのフレームワークは使わず、HTML/CSS/Vanilla JS のみで構築
- 配色は白ベース＋落ち着いた紺アクセント。長時間見ても疲れないことを優先
- PC閲覧をメインに、スマホでも崩れない最低限のレスポンシブ対応
- 長文テンプレは折りたたみ初期状態で、概要だけが見えるようにしている
