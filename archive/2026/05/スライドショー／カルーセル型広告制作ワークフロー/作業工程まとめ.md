# スライドショー／カルーセル型広告制作ワークフロー

## 目的

次回以降、別サービスのスライドショー／カルーセル型広告を作るときに、何から考え、何を決め、Claude Code・Codexをどう使うか迷わないようにするための作業メモ。

基本方針は、先に広告の設計を固めてからClaude Codeに実装させ、最後にCodexでレビューする流れ。

## 全体の流れ

1. 課題・レギュレーションを確認する
2. 広告するサービスを決める
3. ターゲットを決める
4. ターゲットの悩みを決める
5. サービスの強みを決める
6. メインコピーとCTAを決める
7. デザイン方針を決める
8. カルーセルの枚数と流れを決める
9. Claude Code用の実装指示を作る
10. Claude CodeでHTML/CSS/JavaScriptを実装する
11. 表示を確認する
12. Codex用のレビュー指示を作る
13. Codexでコード・広告表現・UIをレビューする
14. 必要に応じて修正する
15. 最終提出用に画面録画・スクショ・説明文を整える

## 1. 最初に確認すること

まず、課題や制作条件を確認する。

確認する項目：

- 広告の種類
  - スライドショー型
  - カルーセル型
  - インタラクティブ広告
- 表示比率
  - 16:9
  - 1:1
  - 縦長
- 使用技術
  - HTML
  - CSS
  - JavaScript
- 必須要素
  - ロゴ
  - CTA
  - スライド枚数
  - 自動再生
  - ナビゲーション
  - 背景画像
  - キャッチコピー
- 禁止・注意事項
  - 不要な仮文字を残さない
  - 赤背景などテストデザインを残さない
  - 既存レギュレーションに反する構成にしない

今回の基本条件：

- 16:9表示
- HTML/CSS/JavaScript
- スライドショー／カルーセル型広告
- CTAを入れる
- ロゴまたはサービス名を入れる
- 広告全体を中央に配置
- 不要文字を削除
- 最終的にブラウザで確認できる状態にする

## 2. サービス設計で決めること

広告を作る前に、以下を埋める。

- サービス名：
- サービス種別：
- 架空サービスか実在サービスか：
- ターゲット：
- ターゲットの悩み：
- サービスの強み：
- 広告で一番伝えたいこと：
- CTA：
- 広告の雰囲気：
- ビジュアル方針：
- 色の方針：
- 避けたい表現：
- スライド枚数：
- 最終的に作るもの：

例：

- サービス名：楽スル
- サービス種別：AI英会話アプリ
- ターゲット：社会人の英語未経験者
- 悩み：英語を話すのが恥ずかしい
- 強み：AI相手だから間違えても恥ずかしくない
- メインコピー：AI相手なら、英語はもっと気軽に話せる。
- CTA：まずは無料体験
- 雰囲気：信頼感、清潔感、やさしさ
- ビジュアル：大人向けのフラットイラスト
- 色：白、ベージュ、グレージュ系
- 避ける表現：怪しいAI感、コンプレックス煽り
- 枚数：4枚
- 成果物：アニメーション付きカルーセル広告

## 3. コンセプトの作り方

コンセプトは、以下の形にすると整理しやすい。

- 誰に向けたサービスか
- どんな悩みを解決するか
- なぜそのサービスなら解決できるか
- 使った後にどう変わるか

コンセプト文テンプレ：

[ターゲット]でも、[サービスの強み]によって、[悩み]を気にせず始められる。
[サービス名]は、[体験価値]を提供する[サービス種別]。

例：

英語に自信がない社会人初心者でも、AI相手なら気負わず始められる。
「楽スル」は、間違えても恥ずかしくない環境で、英会話を楽しく続けられるAI英会話サービス。

## 4. コピー設計

広告では、コピーを増やしすぎない。

決めるコピー：

- サービス名
- メインコピー
- 補助コピー
- スライドごとの見出し
- CTA

メインコピーの考え方：

- 1文でサービス価値が伝わる
- ターゲットの悩みに対応している
- 難しい言葉を使わない
- 煽りすぎない
- 広告の最初に見ても意味がわかる

CTAの考え方：

- 行動が明確
- 心理的ハードルが低い
- サービス内容と合っている

CTA例：

- まずは無料体験
- 無料で試す
- 今すぐ始める
- 詳細を見る
- 5分だけ試してみる
- アプリを見てみる

## 5. スライド構成の基本

4枚構成が扱いやすい。

基本構成：

1. 悩み共感
2. 解決提示
3. サービス価値・機能紹介
4. CTA

各スライドの役割：

- 1枚目：ターゲットの悩みに共感する
- 2枚目：サービスによる解決を見せる
- 3枚目：具体的な機能やメリットを見せる
- 4枚目：行動を促す

注意点：

- 1スライド1メッセージにする
- 文字を詰め込みすぎない
- 1枚目で商品説明をしすぎない
- 最後は必ずCTAを目立たせる
- スライドの流れにストーリーを作る

## 6. デザイン方針の決め方

デザインは、サービス内容とターゲットに合わせる。

決める項目：

- 色
- 雰囲気
- フォント感
- イラストか実写か
- UI中心か人物中心か
- 余白の広さ
- CTAの目立たせ方
- アニメーションの強さ

確認すること：

- ターゲットに合っているか
- サービスの信頼感を下げていないか
- 子どもっぽすぎないか
- 怪しく見えないか
- CTAが埋もれていないか
- スクショで見ても広告として成立するか

今回のような信頼感重視の広告では、以下が合いやすい。

- 白ベース
- ベージュ、グレージュ、アイボリー
- ネイビーやダークグレーの補助色
- 余白多め
- 角丸カード
- 柔らかいシャドウ
- フラットイラスト
- Apple風のミニマルUI

## 7. 合わない組み合わせのチェック

制作前に、方向性がズレていないか確認する。

チェック例：

- 社会人向けなのに、子どもっぽすぎないか
- 信頼感重視なのに、派手すぎないか
- AIサービスなのに、怪しい近未来感が強すぎないか
- 初心者向けなのに、コピーが強く煽りすぎていないか
- イラスト系なのに、サービス内容が軽く見えすぎていないか
- 白ベースなのに、画面が単調になりすぎていないか
- CTAがあるのに、どこを押せばいいかわかりにくくないか

## 8. AIの役割分担

ChatGPT：
- コンセプト整理
- ターゲット整理
- コピー案作成
- Claude Code用プロンプト作成
- Codex用レビュー指示作成
- 作業工程の整理

Claude Code：
- HTML/CSS/JavaScriptの実装
- ファイル構造作成
- アニメーション実装
- カルーセル機能実装
- README作成
- 見返し用HTML作成

Codex：
- コードレビュー
- バグ確認
- アクセシビリティ確認
- レスポンシブ確認
- 広告として弱い点の指摘
- 修正案の提示

使い分け：

- 企画を固める → ChatGPT
- 実装する → Claude Code
- 実装後に厳しく見る → Codex
- Claude Codeが使えない場合の実装代替 → Codex

## UI方針

- 白ベース
- 読みやすいドキュメントUI
- サイドナビまたは目次を入れる
- カード型レイアウト
- 長いテンプレは折りたたみ表示にする
- コピーしやすいボタンを付ける
- スマホよりPCでの閲覧を優先
- 余白を広めに取る
- 見出し階層をわかりやすくする

## 機能

- 目次クリックで該当セクションへ移動
- テンプレごとにコピー用ボタン
- 長いテンプレは開閉できる
- ページ上部に戻るボタン
- 可能なら検索欄を付ける

- 学習ノート・作業マニュアルとして見やすいデザイン
- 派手すぎない
- 長時間見ても疲れない

## 14. 重要な考え方

広告制作では、最初から実装に入らない。

先に決めるべきなのは、誰に、何を、なぜ伝えるか。

実装はClaude Codeに任せられるが、コンセプトが弱いと、見た目だけ整った広告になる。
逆に、コンセプト・ターゲット・悩み・CTAが明確なら、AIに渡す指示の精度が上がり、完成物も安定する。

基本は、

ターゲット → 悩み → 解決 → 機能 → CTA

の順番で考える。

今回の「楽スル」では、

社会人初心者 → 英語を話すのが恥ずかしい → AI相手なら安心 → 5分から楽しく練習 → まずは無料体験

という流れで整理した。
