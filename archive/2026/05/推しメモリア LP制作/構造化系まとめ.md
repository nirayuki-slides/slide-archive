\# 顔出ししないネット発クリエイター向け  

\# デジタル記念カードサービス企画書



\## 1. 企画の結論



本企画は、顔出ししないネット発クリエイター向けに「ファンとの記念体験をデジタルカードとして残せるサービス」として設計するのが最も現実的です。



単なる画像生成ツールではなく、誕生日、周年、配信参加、ライブ記念、ファン会員証などを「推し活の記録」として残せる体験に価値があります。



MVPでは、機能を広げすぎず「推し記念カード生成・配布サービス」に絞るべきです。最初からNFT、ランキング課金、複雑なファンコミュニティ機能に寄せると、企画の焦点がぼやけ、権利面・運用面のリスクも大きくなります。



| 判断項目 | 結論 |

|---|---|

| 最初に作るべきもの | 推し記念カード生成サービス |

| 主な利用者 | 顔出ししない個人VTuber、VSinger、歌い手、配信者 |

| ファン側の価値 | 推し活の記念をスマホに残せる、SNSで共有できる |

| クリエイター側の価値 | 記念企画を簡単に作れる、ファンとの接点を増やせる |

| 差別化の軸 | 販売ではなく「記念体験化」 |

| 注意すべきリスク | 権利管理、無断転載、過度な課金誘導、NFT依存 |

\## 2. 市場背景



推し活市場では、グッズ購入、誕生日企画、周年記念、配信参加、SNS共有など、ファンが「好き」を形に残す行動が広がっています。



特に顔出ししないネット発クリエイターは、実物グッズやイベント開催に比べて、デジタル上で完結する記念コンテンツとの相性が高いと考えられます。



ただし、BOOTHやFANBOXなど既存サービスはすでに強いため、単なる販売場所や月額支援場所として戦うのは弱いです。狙うべきは、販売後・支援後の「記念体験」「SNS共有」「推し活ログ」「企画ページ化」の領域です。



| 項目 | 内容 | 位置づけ |

|---|---|---|

| BOOTH | デジタル・物理グッズを販売できる場所 | 販売には強い |

| FANBOX | 月額支援、限定投稿に強い | 継続支援には強い |

| 本企画 | 記念カード、参加証、会員証を作成・配布 | 記念体験に特化 |

| 差別化余地 | SNS共有、推し活ログ、企画ページ化 | 既存サービスの周辺価値 |



\### 事実・推測・考察の整理



| 区分 | 内容 |

|---|---|

| 事実 | BOOTHは販売場所、FANBOXは月額支援場所として利用されている |

| 事実 | VTuber、VSinger、歌い手、配信者など、顔出ししない活動者が存在する |

| 推測 | 顔出ししないクリエイターは、デジタル記念品との相性が高い |

| 考察 | 販売・支援ではなく、記念体験化に絞ることで差別化しやすい |

\## 3. ターゲット



メインターゲットは、登録者1,000人前後から数万人規模の個人クリエイターです。大手ではなく、記念企画をやりたいが制作・運用の負担が大きい層を狙います。



| ターゲット | 具体例 | ニーズ |

|---|---|---|

| 個人VTuber | 誕生日、活動周年、配信記念を行う人 | ファン向け記念企画を簡単に作りたい |

| VSinger | 歌枠、ライブ、記念配信を行う人 | ライブ参加証や限定カードを配りたい |

| 歌い手 | 投稿記念、誕生日企画を行う人 | SNSで拡散されやすい記念品を作りたい |

| 顔出ししない配信者 | ゲーム配信、雑談配信など | 顔を出さずにファンとの接点を作りたい |



\### 優先ターゲット



| 優先度 | 対象 | 理由 |

|---|---|---|

| 高 | 個人VTuber | 記念日文化、ファンアート文化、SNS共有との相性が高い |

| 高 | VSinger・歌い手 | ライブ・投稿記念・誕生日企画と相性が良い |

| 中 | 小規模配信者 | 需要はあるが、課金意欲や運用頻度の検証が必要 |

| 低 | 大手企業所属クリエイター | 既存の制作体制があり、初期MVPでは狙いにくい |

\## 4. ユーザー課題



このサービスでは、クリエイター側とファン側の課題を分けて考える必要があります。



| ユーザー | 課題 | サービスで解決すること |

|---|---|---|

| クリエイター | 記念企画を作る手間が大きい | テンプレートから短時間でカードを作れる |

| クリエイター | デザイン制作が苦手 | 画像・文言・色を入れるだけで形にできる |

| クリエイター | ファンに何を配ればよいか迷う | 誕生日、周年、配信参加証など用途別に選べる |

| クリエイター | BOOTH販売だけでは記念感が弱い | 企画ページや記念カードで体験化できる |

| ファン | 推し活の記録を残したい | スマホに保存できるカードを受け取れる |

| ファン | SNSで推し活を共有したい | 共有しやすい画像形式で投稿できる |

| ファン | 限定感が欲しい | 期間限定・参加者限定カードを受け取れる |

\## 5. サービスコンセプト

顔出ししないクリエイターの記念日を、ファンと一緒に残せるデジタル記念カードサービス。



\### サービス名候補



| 名称 | 評価 |

|---|---|

現時点では、企画書・LP上では「推しメモリア」を仮名にするのが分かりやすいです。英語名の「Oshi Memory」はロゴやサブ表記として使うと相性が良いです。

| 対象 | 提供価値 |

|---|---|

| クリエイター | 記念企画を簡単に作れる |

| ファン | 推し活の記録を残せる |

| SNS上 | 投稿・拡散しやすい形にできる |

\## 6. MVP機能



MVPでは「カードを作る」「配る」「保存・共有する」に絞ります。



| 優先度 | 機能 | 内容 |

|---|---|---|

| 高 | 記念カード作成 | テンプレートを選び、画像・名前・日付・メッセージを入力 |

| 高 | カード画像生成 | スマホ保存しやすい画像として出力 |

| 高 | 共有用ページ作成 | ファンがカードを受け取るための簡易ページ |

| 高 | SNS共有ボタン | Xなどに投稿しやすくする |

| 中 | 限定公開設定 | 期間限定、URL限定で公開 |

| 中 | ファン名入りカード | ファンが名前を入力して個別カード化 |

| 中 | ボイス付きカード | 音声ファイルを添付できる |

| 低 | 推し活ログ | ファンが過去のカードを一覧保存 |

| 低 | ランキング機能 | 課金競争化しやすいため初期は非推奨 |

| 低 | NFT化 | 権利・理解・導入負荷が高いため初期は非推奨 |



\### MVPで作るべきカード例



| カード種類 | 用途 |

|---|---|

| 誕生日カード | クリエイターの誕生日企画 |

| 活動周年カード | 1周年、2周年などの記念 |

| 配信参加証 | 記念配信、ライブ配信の参加証 |

| ボイス付きカード | 感謝メッセージや限定ボイス付き |

| ファン会員証 | ファンコミュニティ感を出すカード |

| スマホ壁紙カード | 保存・日常利用しやすい記念品 |

\## 9. 価格設計



初期は無料プランと低価格の有料プランで検証するのが現実的です。いきなり高額な月額制にすると、小規模クリエイターには導入されにくい可能性があります。



| プラン | 想定価格 | 内容 |

|---|---:|---|

| 無料プラン | 0円 | 透かしあり、カード作成数制限あり |

| 単発プラン | 300〜800円程度 | 1企画分のカード作成・配布 |

| 月額プラン | 500〜1,500円程度 | 複数企画、透かしなし、テンプレート追加 |

| 上位プラン | 2,000円以上 | ボイス付き、限定ページ、簡易分析など |



\### 価格設計の考え方



| 観点 | 判断 |

|---|---|

| 初期検証 | 無料または低価格で利用障壁を下げる |

| 収益化 | 透かし削除、テンプレ追加、限定公開、ボイス添付で課金 |

| 避けるべき方向 | 投げ銭額ランキング、過度な順位付け、NFT前提の課金 |

| 理由 | 推し活体験より課金競争が前面に出ると印象が悪くなるため |

\## 10. 既存サービスとの差別化



既存サービスと正面から競合しないことが重要です。



| サービス | 強み | 弱み・空白 | 本企画の差別化 |

|---|---|---|---|

| BOOTH | グッズ販売に強い | 記念体験や共有導線は弱い | 購入ではなく記念カード体験にする |

| FANBOX | 月額支援に強い | 単発の記念企画には特化していない | 誕生日・周年など単発企画に強くする |

| X | 拡散に強い | 記念品の作成・保存機能はない | SNS投稿しやすいカードを作る |

| Canva | デザイン作成に強い | 推し活企画専用ではない | 用途別テンプレと配布導線まで用意する |

| 画像生成AI | 画像作成に強い | 企画・配布・保存体験は弱い | 画像生成ではなく記念体験化に絞る |



\### 差別化の核



「作る」だけでなく、「配る」「受け取る」「保存する」「共有する」までを一つの体験にすることです。

\## 11. リスクと対策



| リスク | 内容 | 対策 |

|---|---|---|

| 権利管理 | 他人のイラストや素材を無断使用される可能性 | 利用規約、アップロード時の確認、通報導線を用意 |

| 無断転載 | 公式ではないカードが作られる可能性 | クリエイター本人確認、公式ページ表示、削除申請機能 |

| なりすまし | 他人の名前で企画ページを作られる可能性 | SNS連携や本人確認マークを検討 |

| 課金競争化 | ランキングや金額表示でファン負担が増える | 初期はランキング課金を入れない |

| NFT依存 | 理解されにくく、権利問題も複雑 | 初期MVPでは扱わない |

| デザイン品質 | 安っぽいカードになると価値が下がる | テンプレートの品質を最優先する |

| 利用頻度 | 記念日以外に使われにくい可能性 | 配信参加証、月次カードなど用途を増やす |



\### 弱い部分の指摘



現時点で一番弱いのは、「誰が最初にお金を払うのか」がまだ曖昧な点です。



ファン課金にするのか、クリエイター課金にするのか、無料配布を前提にするのかでサービス設計が変わります。MVPでは、まずクリエイターが企画作成費として少額を払うモデルを優先して検証するのが分かりやすいです。



次に弱いのは、「カードを受け取った後の継続利用」です。保存して終わりになる可能性があるため、SNS共有や過去カード一覧など、再訪する理由を後から追加できる設計にしておく必要があります。

\## 企画としての最重要ポイント



この企画は、機能を増やすよりも「記念体験として分かりやすいか」が重要です。



画像生成、NFT、ランキング、ファンコミュニティなどに広げすぎると、何のサービスか分かりにくくなります。初期は「推しの記念カードを簡単に作って配れる」という一点に絞るべきです。



特にLPでは、サービス説明よりも先に「クリエイターの記念企画が楽になる」「ファンが推し活の記録を残せる」という価値を見せる必要があります。

\## 1行要約



顔出ししないネット発クリエイター向けに、誕生日・周年・配信参加などの記念体験をデジタルカードとして作成・配布できるサービスとして設計する。
